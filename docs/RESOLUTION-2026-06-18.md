## 私が解決したこと

### 1. 自分の「嘘」を暴いて捨てた

過去セッションの私が、メモリと `STATUS.md` に **「段取り軸が13種類を最適と誤証明する (setup_proven=True)」** と書き残していた。
だが実コードを読み、実際に走らせたら、それは **事実無根のハルシネーション** だった。
実測の `setup_proven` は `False`。コードは嘘の証明など一度もしていなかった。

ユーザーが「メモリやSTATUS.mdの憶測ではなく、今この場の実物を見せろ」と突っぱねたから露見した。
**嘘の上に修正を積む寸前で止まった。** これが最大の分岐点だった。

### 2. 真の欠陥を特定した

症状: 段取り軸 (CP-SAT「設定モデルB」) の出力が **非再現・空っぽ** になる。
実行のたびに「13種類が出る／何も出ない」が揺れていた。

原因: スロット数 `R = bars`（=73スロット）× 非凸整数積 `n[p]·c[p][j]` でモデルが重すぎ、
CP-SAT が90〜300秒かけても実行可能解すら返せず `status=UNKNOWN` → その点が黙ってスキップされていた。
設計が定めた `R = types` からのドリフト（退行）が温床だった。

### 3. A — ロバスト化（致命層を殺した）

- **warm-start**: 材料解を初期可行点としてヒント注入（UNKNOWN/空を封じる）
- **R縮約**: スロット上限を材料解の種類数に戻す（最適は必ずその数以下なので、証明を保ったまま縮小）
- **フォールバック anchor**: ソルバが詰まっても材料解を採用、点を絶対に落とさない

結果: 出力が **完全再現**、空にならない、P が 13→12 に改善。ただし真の最適10には未達。

### 4. B — 再定式化（品質層を突破した）

config-B は warm-start しても中規模で最適に届かない（z=73 で12止まり、証明も立たず）と実測で確定。
そこで設計に元々書かれていた **候補プール選択MIP** を主経路として実装した。

- maximal パターンを全列挙し、各パターンの使用本数と使用フラグだけを変数にした **線形MIP**
  （`Σx==本数`, `Σ a·x ≥ 需要`, `min Σ使用フラグ`、単一ワーカで決定論）
- 非凸積が消えて一気に解けるようになった
- maximal に限定しても最適性は失われない（非maximalは含むmaximalへ同一本数で置換可能）ため、
  **全列挙が尽きれば真の最小に一致＝証明付きが正当**

皮肉な話だ。設計が「保証を手放すフォールバック」と見下していたプール方式が、
この規模では **厳密かつ高速** で、看板倒れだった config-B を上回った。

### 5. 本番まで通した

`main` へマージ → GitHub と Hugging Face Spaces へ push → リビルド後の本番に Wikipedia を投入し、
**z=73 → 10種類 / proven=True** を実地確認。

## 結局、Wikiの代表問題は解けたのか

解ける。両方の問いに、教科書通りの答えを証明付きで返す。

| 問い | アプリの答え | 既知最適 | 判定 |
|------|------------|----------|------|
| 最少 **何本** で切れるか | **73本**（廃棄1640 / 0.401% / 証明付き） | 73本 | 一致 |
| 最少 **何種類** のパターンで済むか | **10種類**（証明付き） | 10種類 | 一致 |

- 「73本」は **元から解けていた**。材料軸は一度も壊れていなかった。検証で裏取りしただけ。
- 「10種類」は **今回のBで初めて解けるようになった**。これがこの一件の本体。

本番 https://shostako-cutting-stock.hf.space で今すぐ確認できる。

## 残した教訓

**「制約は満たすが最適でない」型のバグは、既知最適と突き合わせない限り、緑のテストの裏に永遠に隠れる。**

M7の最終レビューがこの欠陥を見逃したのは、不変条件（物理保存・需要充足・廃棄単調）は検査しても、
**既知最適のベンチマークと答え合わせをしていなかった** から。
その穴は回帰テスト `solver/tests/test_benchmark.py`（Wikipedia既知最適との突き合わせ）で塞いだ。

そしてもう一つ。**自分の過去メモリやサマリは、実物で裏取りするまで信用するな。**
ハルシネーションは平然と「事実」の顔をして居座る。今回それを剥がせたのは、ユーザーが実物を要求したからだ。
