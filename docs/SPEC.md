# 仕様 — 1次元カッティングストック最適化

> **2026-06-20 注**: 段取り軸（パターン種類最小化）は本リポから分離し、姉妹プロジェクト `pattern-stock` へ移した。
> 以下の段取り軸 / パレート前線に関する記述は分離前の履歴。本リポの現スコープは材料軸（使用本数最小）のみ。

## 問題定義

1次元カッティングストック問題（1D-CSP）。長い原材料（棒・ロール）から、指定された長さの材料を指定本数だけ切り出す。
切り出しの総コストを最小化する切り方（カットパターンの組み合わせ）を求める。

## 入力

### 初期設定（ツール側で設定、頻繁には変えない）
- **原材料長 L**: 1種類または複数種類（複数長対応は段階的に。まず単一長から）
- **カット代（kerf / カット幅）k**: 1回のカットで失われる幅。切るたびに k だけ消費する。

### パラメータ（実行のたびに変える）
- **需要**: `(材料長 ℓ_i, 必要本数 d_i)` のリスト

## カット代の扱い（重要）

原材料1本に長さ `ℓ_1, ℓ_2, …, ℓ_m` を並べて切る場合、カット代を含めた占有長は概ね:
`Σ ℓ_j + (カット本数) × k ≤ L`
カット本数の数え方（端材側のカットを含めるか、m本取るのに m 回か m−1 回か）は **下記「カット代の数え方（確定）」で Model A に確定済み**。

## カット代の数え方（確定）

採用モデル: **Model A — 各取りに1カット（m本取るのに m カット）**

- 原材料1本に長さ `ℓ_1, …, ℓ_m` を並べて切るとき、各ピースを1カットで切り離す。
  m本目のカットが末尾ピースを残材から分離する。残材（スクラップ）は端に出る。
- **占有長制約**: `Σ_j ℓ_j + m·k ≤ L`
- **ナップサック表現**: 実効幅 `w_j = ℓ_j + k`、ビン容量 `= L`
  （pricing subproblem は「実効幅 `ℓ_j + k`・容量 `L` のナップサック」に素直に畳める）
- **残材（廃棄）** = `L − Σ_j ℓ_j − m·k`（このパターンで1本使ったときの端材）

採用理由:
- 残材が出る一般ケースで物理的に正確（端から1本ずつ切り落とし、最後のカットで末尾ピースを残材から分離）。
- `k` を過小評価しない保守側。1本に詰めすぎて物理的に入らない解を出さない。
- 実効幅 `ℓ_j + k`・容量 `L` の素直な定式化で、列生成のナップサックがクリーンになる。

検算例: `L=2995, ℓ=990, k=10`
- 実効幅 `ℓ+k = 1000` → 1本に入る本数 `m = floor(2995 / 1000) = 2`
- ピース長計 `Σℓ = 2·990 = 1980`、カット代計 `2·10 = 20`、占有計 `= 2000 ≤ 2995` ✓
- 残材 `= 2995 − 1980 − 20 = 995`

不採用モデル:
- **Model B**（`m−1` カット, `Σℓ + (m−1)·k ≤ L`）— 末尾の分離カットを数えない。端境ケースで kerf 1本分だけ緩い。
- **Model A'**（`m+1` カット, `Σℓ + (m+1)·k ≤ L`）— 先頭に面取り（facing）カットを足す。クリーンな基準端が要る工程向け。今回の既定外。

## 目的関数（多目的・トレードオフ）

数学的事実を先に固定する:

- **原材料長が一律なら「使用本数最小」と「廃棄量最小」は同一目的**。総需要長は固定だから、廃棄量 = 使用本数 × L − 総需要長 で本数の単調関数。→ 別物として実装しない。
- **複数原材料長の場合のみ**、本数最小と廃棄量最小は分離する。

本当のトレードオフはこちら:

1. **材料最適**: 使用本数（=廃棄量）の最小化。1D-CSPの主目的。
2. **段取り最少**: 異なるカットパターンの**種類数**の最小化（setup / pattern minimization）。NP困難で、材料最適性と真っ向から競合する。原材料を1本余計に使えば段取りが激減する解が存在する。

→ **パレート前線上の複数解を提示する**のがこのツールの目玉。「材料最優先」「段取り最優先」「中間」を切り替え可能にする。

## 出力・可視化

- 各原材料1本ごとのカットパターンを**棒グラフ状の画像**で表示（どの位置でどの長さを切るか、端材がどれだけ出るか）。
- 解ごとに: 使用本数、総廃棄量、廃棄率、パターン種類数（段取り回数）。
- パレート前線（材料 vs 段取り）を一覧/切り替えで見せる。

## 品質目標

- **最適解 or それに準ずる解**。少なくとも厳密最適とのギャップが評価できること。
- 1D-CSPは整数ギャップが小さい（round-up property）。LP緩和の切り上げで実用上ほぼ最適に届くケースが多いが、**最適性は既知ベンチマークで裏取りする**こと（自己申告で「最適」と言わない）。

## 非機能

- 入力を変えて何度も使う再利用ツール。CLI/GUI両対応を見据え、ソルバ核はI/Oから分離。
