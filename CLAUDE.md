# cutting-stock — 1次元カッティングストック最適化ツール

## このプロジェクトは何か

1次元カッティングストック問題（1D Cutting Stock Problem）を**辞書式最適化**で解く**再利用ツール**。
① 使用本数を最小化（材料最適・arc-flow MIP の gap=0 証明）→ ② その最少本数のまま切り方（パターン種類）の数を最小化
（CP-SAT 候補プール選択MIP・証明付き）。本数も廃棄も犠牲にせず、最少本数の中で最も切り方の少ない計画を**1つ**返す。
原材料長とカット代を初期設定し、欲しい材料長と本数をパラメータで受け取って解く。
リッチなWeb GUIでカットパターンを棒グラフ可視化し、2段の最適性証明バッジ付きで提示する。現場向けのカット指示書（印刷/CSV）も出力する。

> **設計履歴**: 一時は段取り軸を分離して材料専用にし `pattern-stock` を作ったが、切り方最小に現場価値があると判明し
> 撤回。トレードオフ/パレートUIは復活させず、**切り方最小を「材料最適本数を固定した第2目的」として辞書式に内包**する形で
> cutting-stock 単体に統合した（pattern-stock は削除）。二軸パレートUIの旧実装は git 履歴 `f66f8d2` に残る。

## 最初に読め（このセッションが健忘症で始まった場合）

このプロジェクトは別セッション（labで設計）からファイル経由で引き継がれている。
作業を始める前に必ず以下を読むこと:

3. `docs/GUI_DESIGN.md` — **GUI の確定設計（SSOT）**。採用デザイン・実装済み範囲・M5残作業・起動/スクショ手順
4. `docs/PLAN.md` — ビルド方針・技術スタック・ウルトラコードを撃つ箇所
5. `PROGRESS.md` — 現在の到達点と次の一手

## 技術スタック（決定済み）

- **ソルバ核**: Python 3.12 + uv。**辞書式2段**=① Arc-flow MIP on HiGHS（`highspy`）で本数最小を gap=0 厳密に解く → ② CP-SAT 候補プール選択MIP（`ortools`）で本数固定のまま切り方最小を証明（詳細 `docs/SOLVER_DESIGN.md`）。`oracle.py` も独立 CP-SAT で本数を裏取り（CI保険）。GUIから完全分離＝後でCLI/デスクトップ/バッチに載せ替え可能。
- **GUI**: Web（React + Vite）。Python核とはローカルAPIで接続。トレードオフ/パレートUIは持たない（答えは単一解）。
- ディレクトリ: `solver/`（Python核）, `web/`（フロント）, `docs/`（仕様）
- **注**: `docs/` の SSOT には旧・二軸パレートの記述が残るが、現状はパレート掃引せず辞書式単一解。`solve()` が本体。
