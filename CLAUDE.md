# cutting-stock — 1次元カッティングストック最適化ツール

## このプロジェクトは何か

1次元カッティングストック問題（1D Cutting Stock Problem）の**材料最適**（使用本数最小）を厳密に解く**再利用ツール**。
原材料長とカット代（カット幅）を初期設定し、欲しい材料長と本数をパラメータで受け取って解く。
リッチなWeb GUIでカットパターンを棒グラフ可視化し、最適性証明（arc-flow MIP の gap=0）付きで提示する。現場向けのカット指示書（印刷/CSV）も出力する。

> **2026-06-20: 段取り軸（パターン種類最小化）を分離した。** パターン種類の最小化は本数を外部入力に取る別目的のため、
> 姉妹プロジェクト `~/ClaudeCode/pattern-stock`（材料軸＋段取り軸の両方を保持）へ切り出した。本リポは材料最適専用。

## 最初に読め（このセッションが健忘症で始まった場合）

このプロジェクトは別セッション（labで設計）からファイル経由で引き継がれている。
作業を始める前に必ず以下を読むこと:

1. `docs/SPEC.md` — 問題仕様・目的関数・可視化要件・カット代の数え方（Model A 確定）
2. `docs/SOLVER_DESIGN.md` — **ソルバ核の確定設計（SSOT）**。アーキ・アルゴ・データモデル・API・実装マイルストンM0-M7
3. `docs/GUI_DESIGN.md` — **GUI の確定設計（SSOT）**。採用デザイン・実装済み範囲・M5残作業・起動/スクショ手順
4. `docs/PLAN.md` — ビルド方針・技術スタック・ウルトラコードを撃つ箇所
5. `PROGRESS.md` — 現在の到達点と次の一手

## 技術スタック（決定済み）

- **ソルバ核**: Python 3.12 + uv。**材料軸=Arc-flow MIP on HiGHS（`highspy`）** で使用本数最小を gap=0 厳密に解く（詳細 `docs/SOLVER_DESIGN.md`）。`oracle.py` は独立 CP-SAT（`ortools`）で本数を裏取りする検証保険（段取り軸とは別物・CI用途）。GUIから完全分離＝後でCLI/デスクトップ/バッチに載せ替え可能。
- **GUI**: Web（React + Vite）。Python核とはローカルAPIで接続。
- ディレクトリ: `solver/`（Python核）, `web/`（フロント）, `docs/`（仕様）
- **注**: `docs/` の SSOT には段取り軸/パレートの記述が残るが分離前の履歴。現スコープは材料軸のみ（段取り軸は `pattern-stock`）。
